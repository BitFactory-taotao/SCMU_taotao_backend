# 消息模块与交易模块 Postman 测试文档（含 test-only Token）

## 1. 测试目标

验证新增消息模块与交易模块在 `test` 环境下可稳定运行，覆盖：

- 消息模块：会话列表、聊天详情、发送消息、会话已读、全部未读清空、发起交易
- 交易模块：撤回交易、拒绝交易、确认交易
- 鉴权链路：使用 `test-only` 接口生成多身份 token 并执行越权/未登录测试

## 3. 接口概览

### 3.1 test-only token 接口

#### 1）生成指定用户 token（仅 test profile）
- `POST /test-only/auth/token`
- Header：`X-Test-Secret: {{testSecret}}`
- Body：

```json
{
  "userId": "buyer001"
}
```

### 3.2 消息模块接口

- `GET /messages?page=1&size=10`
- `GET /messages/{chatId}?page=1&size=10`
- `POST /messages/goods/{goodsId}/contact`
- `POST /messages/{chatId}`
- `POST /messages/{chatId}/media`
- `POST /messages/{goodsId}/trade`
- `PUT /messages/unread/clear`
- `PUT /messages/{chatId}/read`

### 3.3 交易模块接口

- `DELETE /trade/request/{goodsId}`
- `POST /trade/reject/{goodsId}/{buyerId}`
- `POST /trade/confirm/{goodsId}/{buyerId}`

| `otherToken` | *(自动写入)* | 无关用户 token |

## 7. 业务联调顺序（完整）

1. 调 `test-only` 接口生成 `buyerToken/sellerToken/otherToken`
2. 买家点击“联系商家”：`POST /messages/goods/{goodsId}/contact`
3. 买家：`GET /messages`
4. 买家：`GET /messages/{chatId}`
5. 买家：`POST /messages/{chatId}` 发送消息
6. 买家：`POST /messages/{chatId}/media`（图片/语音消息）
7. 买家：`PUT /messages/{chatId}/read`
8. 买家：`PUT /messages/unread/clear`
9. 买家：`POST /messages/{goodsId}/trade`
10. 卖家：`POST /trade/reject/{goodsId}/{buyerId}`（拒绝流）
11. 买家再次发起：`POST /messages/{goodsId}/trade`
12. 卖家：`POST /trade/confirm/{goodsId}/{buyerId}`（确认流）
13. 买家：`DELETE /trade/request/{goodsId}`（撤回流单独验证）
14. 无关用户/未登录 token 执行越权与未授权测试

## 8. 关键接口示例（可直接复制）

### 8.1 买家获取会话列表

```http
GET {{host}}/messages?page=1&size=10
Authorization: Bearer {{buyerToken}}
```

预期：`200`，`data.list` 非空（有会话时）。

### 8.2 买家联系商家并获取会话

```http
POST {{host}}/messages/goods/{{goodsId}}/contact
Authorization: Bearer {{buyerToken}}
```

预期：`200`，`data.chatId` 存在，`data.sellerId` 与商品商家一致，`data.created` 表示是否新建会话。

### 8.3 买家发送消息

```http
POST {{host}}/messages/{{chatId}}
Authorization: Bearer {{buyerToken}}
Content-Type: application/json

{
  "content": "buyer-msg"
}
```

预期：`200`，`data` 中含消息 ID/发送时间。

### 8.3.1 买家发送图片消息（最小）

```http
POST {{host}}/messages/{{chatId}}/media
Authorization: Bearer {{buyerToken}}
Content-Type: application/json

{
  "contentType": "IMAGE",
  "content": "看下这张图",
  "mediaUrl": "https://example.com/mock/test-image.jpg"
}
```

预期：`200`，`data.id`、`data.sendTime` 存在。

### 8.3.2 买家发送语音消息（最小）

```http
POST {{host}}/messages/{{chatId}}/media
Authorization: Bearer {{buyerToken}}
Content-Type: application/json

{
  "contentType": "AUDIO",
  "content": "",
  "mediaUrl": "https://example.com/mock/test-audio.mp3",
  "mediaDuration": 8
}
```

预期：`200`，`data.id`、`data.sendTime` 存在，`mediaDuration` 为正整数。

### 8.4 买家发起交易

```http
POST {{host}}/messages/{{goodsId}}/trade
Authorization: Bearer {{buyerToken}}
```

预期：`200`，返回 `expireTime`。

### 8.5 卖家拒绝交易

```http
POST {{host}}/trade/reject/{{goodsId}}/{{buyerId}}
Authorization: Bearer {{sellerToken}}
```

预期：`200`，消息类似 `已拒绝该交易请求`。

### 8.6 卖家确认交易

```http
POST {{host}}/trade/confirm/{{goodsId}}/{{buyerId}}
Authorization: Bearer {{sellerToken}}
```

预期：`200`，`data.tradeId` 存在，商品状态转已交易。

### 8.7 买家撤回交易

```http
DELETE {{host}}/trade/request/{{goodsId}}
Authorization: Bearer {{buyerToken}}
```

预期：`200`，消息类似 `您已成功撤回交易请求`。

## 9. 异常与边界测试清单

### 9.1 鉴权类
- 不带 `Authorization`
- 错误 token / 过期 token
- 买家 token 调卖家接口
- 无关用户 token 调卖家接口
- `X-Test-Secret` 错误（调用 test-only 应返回 `403`）

### 9.2 参数类
- `chatId=0`
- `goodsId=0`
- `page=0` / `size=0`
- `content=""`
- `content` 长度 `>1000`

### 9.3 业务类
- 自己给自己发起交易
- 商品不存在
- 商品已成交/已下架
- 重复发起交易（检查是否返回剩余 TTL）
- 撤回后再确认（应失败）
- 过期意向确认（应失败）

---

## 10. 结果判定标准

满足以下条件可判定通过：

1. 正常流程全部返回 `200`
2. 未授权/越权场景返回预期 `401/403/业务失败`
3. 参数校验能正确拦截
4. 消息未读/已读状态正确
5. 交易发起-撤回-拒绝-确认链路行为一致
6. 交易确认后商品状态与交易记录一致

---

## 11. 备注

- test-only 接口只用于测试环境，不用于生产。
- 若 `POST /test-only/auth/token` 返回 `404`，先确认服务是否以 `test` profile 启动。
- 若返回 `403`，优先检查 `X-Test-Secret` 与 `application-test.yaml` 是否一致。
